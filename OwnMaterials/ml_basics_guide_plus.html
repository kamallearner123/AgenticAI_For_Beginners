3) Data preparation

Missing values

Strategies

  • Numerical: mean/median or model‑based imputation.
  • Categorical: new category like “Unknown”.
  • Don’t leak: fit imputer on train only.

Code

from sklearn.impute import SimpleImputer
imp_num = SimpleImputer(strategy="median")  # fit on X_train
imp_cat = SimpleImputer(strategy="most_frequent")
Encoding & scaling

Choose wisely

  • One‑hot for nominal categories.
  • Ordinal for ordered labels.
  • Standardize features for SVM/KNN/logistic.

Pipeline

from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import OneHotEncoder, StandardScaler
pre = ColumnTransformer([("num", StandardScaler(), num_cols),
                         ("cat", OneHotEncoder(handle_unknown="ignore"), cat_cols)])
Feature engineering & selection

Ideas

  • Log transforms for skew.
  • Binning (quantiles).
  • Ratios, interactions.
  • Target‑aware selection via CV.

Permutation importance

from sklearn.inspection import permutation_importance
r = permutation_importance(model, X_val, y_val, n_repeats=10, random_state=0)
print(r.importances_mean)
Data leakage traps
  • Imputing/scaling on full dataset.
  • Using future info in time series.
  • Label encoding with global frequencies.

4) Metrics deep‑dive

Classification

  • Accuracy = (TP+TN)/(TP+FP+TN+FN)
  • Precision = TP/(TP+FP)
  • Recall = TP/(TP+FN)
  • F1 = 2·(P·R)/(P+R)
  • ROC‑AUC vs PR‑AUC (use PR‑AUC for imbalance)

Regression

  • MSE / RMSE, MAE
  • R² (variance explained)
  • MAPE (scale‑free, beware zeros)

Clustering

  • Silhouette score
  • Davies–Bouldin index
  • Stability across time splits

5) Model zoo (when to pick what)

Linear & Regularized

Fast, strong baselines. Use L1/L2 (Lasso/Ridge).

from sklearn.linear_model import Ridge
model = Ridge(alpha=1.0).fit(Xtr, ytr)

Trees & Ensembles

Great on tabular data; capture non‑linearities.

from sklearn.ensemble import HistGradientBoostingRegressor
model = HistGradientBoostingRegressor().fit(Xtr, ytr)

SVM / k‑NN

SVM for margins, k‑NN for lazy learning; scale features.

Naive Bayes

Strong baseline for text (MultinomialNB).

Neural nets

MLPs for tabular; CNNs for images; Transformers for text.

6) Hyperparameter tuning

Randomized search

from sklearn.model_selection import RandomizedSearchCV
from sklearn.ensemble import RandomForestClassifier
param = { "n_estimators":[200,400,800], "max_depth":[None,10,20,30] }
cv = RandomizedSearchCV(RandomForestClassifier(), param, n_iter=6, cv=5, n_jobs=-1, random_state=0)
cv.fit(Xtr, ytr); print(cv.best_params_)

Cross‑validation choices

  • StratifiedKFold for classification
  • GroupKFold for grouped leakage
  • TimeSeriesSplit for temporal data

Early stopping (gbdt)

from sklearn.experimental import enable_hist_gradient_boosting  # noqa
from sklearn.ensemble import HistGradientBoostingClassifier
clf = HistGradientBoostingClassifier(early_stopping=True, validation_fraction=0.2)

7) Imbalanced data

Strategies

  • Use class_weight="balanced"
  • Tune decision threshold
  • Use PR‑AUC, F1, Recall@k
  • Try anomaly detection for extreme skew

Code

from sklearn.linear_model import LogisticRegression
clf = LogisticRegression(class_weight="balanced", max_iter=300).fit(Xtr, ytr)

9) NLP starter

Text pipeline

from sklearn.pipeline import Pipeline
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression
nlp_pipe = Pipeline([("tfidf", TfidfVectorizer(ngram_range=(1,2))),
                     ("clf", LogisticRegression(max_iter=300))])

Tips

  • Lowercase, strip punctuation
  • Try character n‑grams for noisy text
  • Use class weights for imbalance

12) Deployment

Save pipeline

import joblib
joblib.dump(pipe, "model.joblib")
pipe = joblib.load("model.joblib")

REST API (Flask)

from flask import Flask, request, jsonify
import joblib
app = Flask(__name__); pipe = joblib.load("model.joblib")
@app.post("/predict")
def predict():
    X = request.get_json()
    return jsonify(pred=pipe.predict(X).tolist())

Checks

  • Schema & type checks
  • Latency & cost budgets

14) Ethics, fairness & privacy

Checklist

  • Data provenance documented
  • User consent & purpose limitation
  • Fairness: parity of error rates across sensitive slices
  • Privacy: PII handling, aggregation, anonymization

Audits

Run bias & robustness tests pre‑launch; document caveats.

Governance

Versioned artifacts, approvals, escalation paths.

15) Cheat‑sheets

Model selection

  • Small tabular → trees/ensembles
  • High‑dim sparse text → linear/SVM
  • Images → CNN
  • Sequences → RNN/Transformer

Common pitfalls

  • Leakage between train/test
  • Wrong CV for time series
  • Chasing leaderboard overfitting

Quick glossary

  • Regularization: penalties that reduce overfitting
  • Calibration: matching predicted probabilities to reality
  • AUC: probability a random positive ranks above a negative
