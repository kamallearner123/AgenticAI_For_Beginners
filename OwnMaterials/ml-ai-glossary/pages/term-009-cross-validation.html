Cross-Validation

Technique to estimate model generalization by splitting data into folds.

When to use

  • Applies to problems involving evaluation aspects.
  • Prefer this concept/model when its assumptions roughly hold.
  • Validate with a baseline and compare using multiple metrics.

Why it matters

Cross-Validation is commonly used in the Evaluation category. Understanding it helps you make better choices when designing pipelines, evaluating models, and communicating results.

Intuition

Technique to estimate model generalization by splitting data into folds.

Concrete example

Example: 5-fold CV trains on 4 folds and validates on the 5th, rotating 5 times.

Code Snippet

Try this minimal example (python).

# Cross-validation with Pipeline
from sklearn.datasets import load_breast_cancer
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import cross_val_score

X, y = load_breast_cancer(return_X_y=True)
pipe = Pipeline([("scaler", StandardScaler()), ("clf", LogisticRegression(max_iter=200))])
scores = cross_val_score(pipe, X, y, cv=5, n_jobs=-1)
print("CV accuracy:", scores.mean())
